# Contributing

## How Listings Work

All listings are pre-populated by the maintainer using accurate, factual, up-to-date information from public sources (official websites, app stores, documentation, public repos). No information is assumed or inferred.

Platform owners can submit corrections or request new listings through GitHub issue templates. You do not need to submit your own listing from scratch — it may already be there.

---

## Request a Correction

If your platform is already listed and something is inaccurate:

1. Open a new issue using the **Correction** template
2. Specify which fields are incorrect and the correct values
3. Attach a verification screenshot (see below)
4. Maintainer verifies and publishes changes

---

## Request a New Listing

If your platform is not yet listed:

1. Open a new issue using the **New Listing** template
2. Provide the platform name, website URL, and basic details
3. Attach up to 5 app screenshots for product showcase
4. Attach 1 verification screenshot (see below)
5. Submission enters the review queue — no auto-publish
6. Maintainer researches and populates the full schema from public sources, then publishes

---

## Verification

Every correction and new listing request requires a verification screenshot.

- Display the string `discordalt-verify-[platform-name]` somewhere visible inside the running application (e.g., in a chat message, channel name, or settings field)
- Take a screenshot showing the verification string on screen
- Attach the screenshot to your issue

This proves you control the product and prevents impersonation. Verification screenshots are not displayed on the live site.

---

## Screenshot Guidelines

Product showcase screenshots (separate from the verification screenshot):

- Up to 5 screenshots per listing
- Show the actual app UI — not marketing renders or mockups
- PNG or JPEG
- Minimum 1280px wide recommended
- No personal information visible in screenshots

---

## What Happens After Submission

1. Maintainer reviews the issue
2. Verification screenshot is checked
3. For corrections: fields are updated from the provided info and verified against public sources
4. For new listings: maintainer populates the full schema from public sources
5. Changes are committed and deployed

There is no guaranteed timeline. All reviews are done manually.
